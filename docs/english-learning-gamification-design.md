# Lexicon 英文學習成就系統設計

> 狀態：待實作設計稿（2026-07-17）  
> 前提：建立在既有「學這句、今日學習、間隔複習、情境任務」上；所有資料與判定維持本機。

## 1. 目標與邊界

### 目標

讓使用者每天完成一小段真正有效的英文練習後，立刻感受到三件事：

1. **今天有完成**：有清楚、可在 5 分鐘內達成的目標。
2. **自己有進步**：不只累積分數，而是能看見表達從「待熟悉」變成「已能使用」。
3. **明天想回來**：有溫和的連續感與下一個可解鎖的小目標。

### 不做的事

- 不做公開排行榜、聯盟降級、愛心扣光或付費復活；它們會把焦點從學會英文移到避免失敗。
- 不以看答案、重複點擊或收藏量灌水取得進度。
- 不把 streak 當能力指標；中斷時保留學習歷史，且不羞辱使用者。
- 不新增雲端帳號、遠端分析或社交壓力。

## 2. 遊戲核心：能力地圖 + 每日旅程

Lexicon 的遊戲單位不是虛構世界的關卡，而是使用者已收藏的真實溝通能力。每一個「能力節點」對應一個情境與可使用的英文表達，例如：

```text
工作溝通
  └─ 詢問進度
      └─ update someone on ...
          ├─ 收藏：看懂並選擇要學
          ├─ 回想：能自行寫出
          ├─ 變化：能換情境使用
          └─ 實戰：能在任務訊息中使用
```

使用者每天只需完成「今日旅程」：從當天到期題目選出 3 個小節點，預估 3–7 分鐘。完成旅程後獲得 XP、維持連續日、推進本週目標；但若還有動力，仍可自由練習，不增加當日主目標壓力。

## 3. 成就迴路

```text
翻譯或字幕遇到真實需求
  → 收藏一個值得再次使用的表達
  → 今日旅程主動回想
  → 立即回饋與 XP
  → 表達節點升階 / 解鎖任務
  → 在新的情境正確使用
  → 能力地圖留下可見的「我真的會了」
```

### 每日完成條件

「今日完成」以下列任一方式成立，避免單一題型造成阻塞：

- 成功完成 3 個複習題；或
- 成功完成 2 個複習題 + 1 個情境任務；或
- 沒有到期題目時，收藏 1 句並完成它的首次回想。

完成後顯示一句具體回饋，例如「今天把『催進度』從提示回想練到自行使用」，而非只說「+20 XP」。

## 4. XP、等級與熟練度

### XP 規則

| 行為 | XP | 上限與理由 |
| --- | ---: | --- |
| 首次建立學習項目 | 5 | 每日最多 15；鼓勵挑有價值的句子，不鼓勵大量收藏。 |
| 無提示答對複習題 | 10 | 每日最多 50；核心獎勵。 |
| 需提示後修正成功 | 6 | 保留進步感，但低於無提示答對。 |
| 失敗後完成指定重試 | 4 | 鼓勵修正，不懲罰錯誤。 |
| 完成情境任務 | 20 | 每日最多 20；鼓勵遷移使用。 |
| 將項目升為「已能使用」 | 15 | 無每日上限；獎勵真正長期保留。 |

XP 只在儲存有效 review event 後計入；重看答案、切換頁面、重複送出同一事件均不加分。每日有效 XP 上限為 85，避免刷題。

### 等級

使用累積 XP 的平方根曲線：`下一級所需 XP = 40 + 10 × 目前等級`。前幾級很快取得回饋，之後進度平緩。

| 等級 | 名稱 | 代表意義 |
| --- | --- | --- |
| 1–2 | 起步 | 建立自己的第一批表達。 |
| 3–5 | 回想者 | 開始能在隔天自行取回表達。 |
| 6–9 | 應用者 | 能跨情境改寫並完成任務。 |
| 10+ | 表達者 | 累積多個可直接使用的真實能力。 |

等級僅作為旅程總覽，不用來推斷 CEFR 程度或與他人比較。

### 能力節點的四階

這比角色等級更重要，且直接沿用目前學習狀態：

| 階段 | 取得條件 | 視覺狀態 | 對使用者的意義 |
| --- | --- | --- | --- |
| 收藏 | 建立 Learning Item | 空心節點 | 我想學這句。 |
| 回想 | 首次 `good` 或 `easy` | 半亮節點 | 我能在提示少的情況下想起它。 |
| 靈活 | 不同日期、不同題型各成功一次 | 實心節點 | 我能換情境使用。 |
| 已能使用 | 現有 `mastered` 條件或任務成功使用 | 金色星節點 | 下次有機會可以自己說。 |

錯誤只會令該節點暫時回到「需要再練」，不會扣 XP、不會抹去已完成階段。

## 5. 連續日與保護機制

### Streak 定義

- 以使用者本機時區的曆日計算；一天完成一個「今日完成」條件即 +1。
- 新的一天從 00:00 起算；在凌晨補完上一日的題目仍算新一天，規則固定且清楚。
- 顯示「本週完成 4 / 7 天」優先於長 streak，降低斷掉後放棄的風險。

### 旅程保留券

- 每完成 6 個不同日期，獲得 1 張保留券；最多存 2 張。
- 漏掉一天時自動使用 1 張，並明確顯示「幫你保留旅程；今天回來就好」。
- 沒有保留券時 streak 歸零，但「最長連續」與能力地圖不歸零。
- 使用者可在設定停用 streak；停用後仍保有每日旅程與 XP。

## 6. 每日任務與週目標

### 今日旅程畫面

置於「今日學習」頂端，取代目前只有統計的開場：

```text
今日旅程                         Lv. 3 回想者    142 / 170 XP
連續 4 天 · 本週 4 / 7 天                            [保留券 1]

□ 回想 1 個工作表達                  0 / 1
□ 完成 2 個到期複習                  1 / 2
□ 用 2 個表達完成一則情境訊息          0 / 1

完成 2 / 3 項即可完成今日旅程                         [繼續練習]
```

系統依到期項目生成任務；沒有足夠資料時使用新手版本：收藏一句、完成首次回想、查看一個使用提醒。任務文字永遠指出語言行為，而不是「獲取 10 XP」。

### 每週目標

每週一建立，不強迫選擇：

- 預設：「完成 4 天今日旅程」。
- 有 5 個以上「正在變熟」項目時，再加：「讓 1 個表達進入靈活」。
- 有 3 個以上已熟表達時，再加：「完成 1 次情境任務」。

週末只做回顧：完成了什麼、哪種情境更有把握、下一週可專注什麼；不設失敗畫面與倒扣。

## 7. 徽章與解鎖

徽章必須對應可觀察的學習里程碑，一次完成後永久保留。第一版只提供 12 枚，避免變成雜訊。

| 群組 | 徽章 | 條件 |
| --- | --- | --- |
| 啟程 | 第一顆星 | 完成第一個學習項目。 |
| 啟程 | 三日旅程 | 在任意 7 日內完成 3 天。 |
| 回想 | 隔天想起來 | 收藏後隔日首次答對。 |
| 回想 | 不靠提示 | 連續 5 題無提示答對。 |
| 應用 | 換個說法 | 第一次完成 rewrite 題。 |
| 應用 | 真實訊息 | 第一次完成情境任務。 |
| 成長 | 越來越熟 | 第一個項目成為已能使用。 |
| 成長 | 五顆能力星 | 累積 5 個已能使用項目。 |
| 情境 | 工作溝通 | 工作標籤項目累積 3 個已能使用。 |
| 情境 | 聽力線索 | 由 YouTube 字幕建立並答對 3 個項目（Phase 2）。 |
| 自主 | 整理者 | 封存或刪除 5 個不再需要的項目。 |
| 穩定 | 一週回來 | 任意 7 日內完成 5 天旅程。 |

解鎖的內容是實用功能：完成首個「靈活」節點後開放情境任務；有 5 個項目後開放個人能力地圖。不得以付費或隨機抽獎取得。

## 8. 畫面與互動規格

### 導覽

- 側欄名稱維持「今日學習」；完成當日後顯示小型勾號，不顯示煽動性紅點。
- 進入時先看「今日旅程」，再看下一題；能力地圖與徽章作為同頁分區，第一版不新增多一個工作區。

### 答題中

- 題目頂部顯示 `第 2 / 3 個今日目標`，不是倒數計時。
- 正確：顯示 `+10 XP`、節點填亮的短動畫與一句具體能力描述。
- 接近或錯誤：顯示現有精簡修正；重試成功後顯示 `修正成功 +4 XP`。
- 動畫需可在「減少動態效果」設定中關閉；不使用聲效作為唯一回饋。

### 完成畫面

```text
今天完成了
你完成了 3 次主動回想，並讓「update someone on」更接近可直接使用。

+26 XP   ·   連續第 5 天   ·   工作溝通能力地圖 +1

[查看今天的進步]  [再練一題]
```

只在第一次達成每日條件時顯示；之後繼續練習維持平靜的 inline 回饋。

### 新手空狀態

不顯示 0 分或空白地圖。以一段流程教學引導：

1. 在翻譯結果按「學這句」。
2. 選擇真正想下次自己說出的句子。
3. 完成第一個回想題，建立你的第一顆能力星。

## 9. 資料模型與計算

在既有 SQLite 新增下列資料；`review_events` 仍是 XP 的唯一事實來源，避免前端自行累加。

```text
learner_profile
  id (固定 1), total_xp, level, current_streak, longest_streak,
  last_completed_local_date, journey_shields, streak_enabled,
  reduced_motion, updated_at

daily_journeys
  local_date (PK), status (active|completed), required_units,
  completed_units, xp_earned, completed_at

daily_journey_tasks
  id, local_date, kind (review|task|save_first_item), target_count,
  progress_count, completed_at, metadata_json

xp_events
  id, source_type (learning_item|review|task|mastery|badge), source_id,
  amount, local_date, created_at,
  UNIQUE(source_type, source_id, amount)

achievements
  code (PK), unlocked_at
```

重要規則：

- 所有日期以 renderer 傳入的 IANA timezone 轉成本機日期，main process 驗證後寫入；不可直接用 UTC 日期判斷 streak。
- 完成 review 後，同一個 transaction 依序：寫 review event → 判定節點階段 → 寫 XP event → 更新每日任務 → 必要時完成旅程與 streak → 判定徽章。
- `xp_events` 的唯一鍵防止 IPC 重送或重啟造成重複加分。
- 清除學習資料同時清除以上遊戲資料；刪除單一 Learning Item 不回收既得 XP 或徽章，但能力地圖不再顯示其內容。

## 10. IPC 與前端契約

新增單一可讀 dashboard，避免 renderer 自己拼湊多個狀態：

```ts
type GamificationDashboard = {
  profile: { totalXp: number; level: number; title: string; currentStreak: number; longestStreak: number; shields: number }
  today: { localDate: string; completed: boolean; requiredUnits: number; completedUnits: number; xpEarned: number; tasks: JourneyTask[] }
  week: { completedDays: number; targetDays: number }
  abilityMap: AbilityNode[]
  achievements: Achievement[]
}
```

- `loadLearningDashboard()` 擴充為同時回傳 `gamification`，讓學習畫面只載入一次。
- `reviewLearningItem()` 的回應擴充 `rewards`：本次 XP、是否節點升階、是否完成任務、是否完成今日旅程、剛解鎖徽章。
- `createLearningFromRecord()` 成功後同樣回傳本次 XP／任務推進，讓「學這句」可立即更新。
- 所有獎勵文字由 renderer 根據結構化結果組裝；main process 只負責判定與持久化。

## 11. 實作拆分與驗收

### Phase G1：可靠的進度核心

1. SQLite migration、profile、XP event 與每日旅程服務。
2. 將既有 review 流程改成 transaction，回傳結構化 reward。
3. 今日旅程計算、完成判定、streak 與保留券。
4. 單元測試：時區跨日、重送冪等、漏日使用保留券、錯誤不扣分、每日 XP 上限。

### Phase G2：可見的成就感

1. `LearningWorkspace` 頂端加入今日旅程、XP、streak 與完成回饋。
2. 在答題回饋中顯示 XP／節點升階；支援 reduced motion。
3. 側欄每日完成勾號。
4. 單元測試加上 Vue 元件狀態測試；手動驗證亮／暗主題。

### Phase G3：長期動機

1. 能力地圖與 12 枚徽章。
2. 週回顧與可關閉 streak／動畫設定。
3. YouTube、IELTS 接入同一個能力節點（僅在既有來源資料可靠後）。

### MVP 驗收

- 新使用者可在三步內取得第一個 XP，且知道 XP 來自哪個英文行為。
- 完成三個有效複習後，只觸發一次今日完成與 streak。
- 同一 review IPC 重送不會重複獲得 XP、徽章或 streak。
- 漏一天時，有保留券才保留 streak；沒有時只重置 current streak。
- 使用者可關閉 streak／動畫；關閉不影響學習排程。
- 所有遊戲資料留在 `%APPDATA%\\Lexicon\\lexicon.sqlite`，不增加網路請求。

## 12. 成功指標與停止條件

新增的成就系統只有在以下同時成立時才算成功：

- 每週完成至少 3 天旅程的使用者比例上升。
- 1 天與 7 天後的無提示正確率不下降。
- 情境任務完成率上升，且收藏後立即放棄率不惡化。
- 使用者沒有因 streak 中斷而大量停止回來（以中斷後 7 天內回訪觀察）。

若 XP 增加但延後正確率、任務完成率沒有提升，先調低獎勵與任務壓力，而不是再加更強的打卡或排行榜機制。
